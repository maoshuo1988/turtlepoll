# 产品概述

开战广场是龟投论坛的核心对战模块，采用 **1v多（庄家制）** 模式。用户可以作为庄家发起赌局，或作为挑战者加入他人的赌局，围绕各类事件（体育赛事、财经走势、AI 预测等）进行龟币押注对战。此系统沿用金币，但是不复用预测市场领域的模型与接口。
# 关键指标
加入/加注/提取/宣布/仲裁都必须幂等
账务流水与对账：必须可追溯（尤其仲裁罚没 10% 的下注）

> 资金模型：下注采用“扣款托管进资金池”，不使用冻结；入场费实时转给庄家；罚没资金统一销毁（burn）。

# 用户角色

| 角色 | 说明 |
|------|------|
| **庄家** | 发起赌局，定义议题、立场、押注金额、结算时间，负责宣布结果 |
| **挑战者** | 加入赌局，持与庄家相反立场，选择押注金额 |
| **管理员** | 处理争议仲裁（最终裁定，不可上诉） |

# 核心流程

## 赌局生命周期

```
创建赌局 → 等待挑战者 → 封盘 → 结算时间到达 → 庄家宣布结果 → 挑战者确认/异议 → 结算完成
```
## 赌局
每个赌局记录自己的庄家以及挑战者信息
挑战者下注信息（允许同一挑战者多次追加下注）
庄家下注信息

## 详细状态

| 状态 | 标识 | 说明 |
|------|------|------|
| 🟢 开放中 | `open` | 接受挑战者加入 |
| 🔒 已封盘 | `sealed` | 不再接受新挑战者（容量满或手动封盘；到达结算时间后将由系统自动进入 `pending`） |
| ⏳ 待宣布 | `pending` | 结算时间已到，等待庄家24h内宣布结果 |
| ✅ 已结算 | `settled` | 结果确认，龟币已分配 |
| ⚠️ 争议中 | `disputed` | 有挑战者提出异议，等待管理员仲裁 |


## 术语

### 入场费
赌局是公开赌局时挑战者投注时扣除的手续费，默认为5%
入场费直接给庄家，不退回

### 资金池（托管）
- **下注本金**（庄家押注、挑战者下注）在发生时会从用户账户扣款并进入“资金池（托管账户）”；结算/作废/退款均从资金池再分发。
- **入场费**不进入资金池：挑战者下注时实时扣款并转给庄家，后续不退。
- **罚没/处罚**统一走“销毁（burn）”：从资金池（或用户应得）划转到销毁账户，形成可审计流水。

### 账务结构（实现约束）
为保证可追溯与幂等，采用：
1) **统一 Ledger（流水）**：记录所有资金动作（入池、出池、入场费转账、销毁）。
2) **结算单（Settlement）**：在 `settled` 时生成每个参与者的应得金额（可提取），提取时按结算单出池并幂等。
3) `battle` 表 **冗余汇总字段** 便于查询：庄家押注总额、挑战者下注总额、资金池本金总额、入场费总额、销毁总额等。


### 结算流程

#### 第一步：庄家宣布结果
- 结算时间到达后，庄家需在 **24小时内** 宣布结果
- 选项：庄家赢 / 庄家输
- **超时未宣布 = 自动判定庄家输**

#### 第二步：挑战者确认
- 每位挑战者有 **24小时** 确认或提出异议
- 沉默（超时未操作）= 默认确认

#### 第三步：结算 / 仲裁
- 全部确认 → 立即结算
- 任一挑战者提出异议 → 进入管理员仲裁（24h内裁定，最终决定不可上诉）

#### 第四步：提取（资金出池）
- `settled` 后不会立刻把资金从资金池打给用户；系统先生成结算单。
- 庄家/挑战者通过“提取接口”领取应得金额，提取动作会根据结算单从资金池出池到用户账户，并落账务流水。

### 后台任务（状态迁移/超时）
系统需通过后台轮巡任务保证流程自动闭环（所有任务必须幂等）：
1) **封盘任务（open → sealed）**：到达结算时间或容量已满/手动封盘则封盘。
2) **到期任务（sealed → pending）**：到达结算时间后自动进入 `pending`，并写入 `pending_deadline = now + 24h`。
3) **庄家超时任务**：`pending` 超过 `pending_deadline` 未宣布，自动记为“庄家输”（进入结算流程）。
4) **挑战者确认超时任务**：挑战者确认窗口超时未操作，记为默认确认。
5) **争议超时任务（disputed → settled, void）**：进入 `disputed` 后写入 `dispute_deadline = now + 24h`，超时未裁定则**默认 void**并结算。

## 创建赌局
创建赌局的用户作为庄家，押注金额必须大于等于100

**表单字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 议题（事件标题） | 文本输入 | ✅ | 赌局主题，需明确可判定 |
| 庄家立场 🔴 | 多行文本 | ✅ | 庄家的预测/观点 |
| 挑战者立场 🔵 | 多行文本 | ✅ | 与庄家相反的立场 |
| 押注金额 | 快捷选择 + 自定义 | ✅ | 必须大于100 |
| 公开/私人 | 切换按钮 | ✅ | 公开：所有人可见；私人：需邀请码加入 |
| 结算时间 | 日期时间选择器 | ✅ | 到达后自动封盘 |

## 加入赌局（挑战者下注）
记录本次赌局加入的挑战者信息，并扣除相应金额：
- 挑战者下注本金：**扣款并托管进资金池**
- 入场费（公开赌局 5%）：**扣款并实时转给庄家**（不进入资金池，不退回）

赌局金额中记录挑战者下注本金（资金池本金的一部分）。

### 追加下注规则（挑战者可多次下注）
- 同一挑战者允许对同一赌局**多次下注**（追加下注）。
- 容量校验、封盘判断、结算分配 **均按挑战者在该赌局的累计下注额** 计算。
- 列表展示“挑战者列表”时：
	- 可返回“挑战者累计下注额（sum）”，以及可选的“下注明细列表”。
- 结算单生成时：以“用户维度汇总后的累计下注额”为准（不按单笔下注逐条结算）。
如果总的挑战者下注总额超过庄家押注额，则拒绝本次下注，并返回最多还能下注多少
挑战者下注总额达到庄家押注额自动封盘
用户如果已经是庄家，则拒绝加入
金币流水记录两类流水：挑战者下注本金入池流水、入场费转给庄家的流水


**加入条件：**
- 赌局状态为"开放中"
- 挑战者下注总额未超过庄家押注额（容量限制）

**费用结构：**

| 赌局类型 | 入场费 | 说明 |
|----------|--------|------|
| 公开赌局 | 5% | 即时转给庄家（不退还） |
| 私人赌局 | 0% | 无入场费 |

**押注选项：** 50 / 100 / 200 / 500 / 1,000 / 2,000 🪙

**容量机制：**
- 所有挑战者下注总额 ≤ 庄家押注金额
- 容量满后自动封盘
- 显示容量进度条（所有挑战者已下注 / 庄家押注额）
- 记录投注以及手续费等金额流水 

## 庄家加注
允许庄家对没有进入结算时间的赌局进行加注，传入赌局ID，增加押注金额
如果加注的赌局已经封盘但是结算时间还未进入，赌局再次开启可以挑战者下注
如果加注的赌局已经封盘但是结算时间已经过了，本次不可以加注



## 后台轮巡封盘
系统每5分钟轮巡一次，检查所有5分钟内未进入结算时间的赌 局，如果已到结算时间，则自动封盘

> 补充：到达结算时间后，系统还需将 `sealed` 自动切换为 `pending`，并启动庄家 24h 宣布倒计时。

## 庄家宣布结果
允许庄家对已封盘的赌局进行结果确认，传入赌局ID，确认结果
如果庄家超时未确认结果，则自动判负

## 挑战者确认/异议
允许挑战者对已封盘的赌局进行异议，传入赌局ID，确认异议

## 管理员裁决赌局结果
管理员可以裁决赌局结果，传入赌局ID，确认结果
进入 `disputed` 后以管理员裁决为最终结果（挑战者确认仅作为证据/背景信息）。

管理员裁决超时策略：超时未裁定则默认 `void`（作废）并进入结算。

| 违规行为 | 仲裁结果 | 处罚 |
|----------|----------|------|
| 庄家虚假宣布结果 | banker_loses | 正常输钱赔付 + 扣除本金 10%的金币（如果庄家裁决后是输，那么从庄家的账户里扣除） + 封号7天 |
| 议题模糊不可判定 | void（作废） | 挑战者全额退还，庄家扣除 10% 本金 + 封号7天（入场费不退） |
| 挑战者虚假异议 | banker_wins | 正常庄家赢赔付 + 扣除异议下注额 10% |

## 提取接口
挑战者/庄家提取已结算的赌局，金币流水记录赌局获得的金币

### 提取规则（一次性全提）
- 每个用户对同一赌局仅允许**一次性全额提取**（不支持部分提取）。
- 提取基于结算单的“应得金额”一次性出池到用户账户。
- 提取必须幂等：同一结算单/同一用户重复请求不会重复出池。
**庄家赢：**
```
庄家收益 = 退还本金 + 所有挑战下注金额 + 已收入场费
```

**庄家输：**
```
每位挑战者收益 = 退还本金 + 庄家押注额 × (我下注额 / 所有挑战者下注总额)
```

### 结算与资金分配（托管资金池口径）
以下描述用于明确“本金退还/赔付”在账务上的落地方式：

1) **庄家赢（banker_wins）**
- 入场费：已在下注时实时转给庄家（不退）。
- 资金池出池：将（庄家押注本金 + 挑战者下注本金）从资金池转给庄家。

2) **庄家输（banker_loses）**
- 资金池出池：
	- 每位挑战者：退还自己下注本金 + 按比例分得庄家押注本金。
	- 庄家：无资金池返还（视为输掉押注本金）。

3) **作废（void）**
- 入场费：不退（已转给庄家）。
- 资金池出池：
	- 挑战者：全额退还各自下注本金。
	- 庄家：退还庄家押注本金的 90%。
	- 处罚 10%：从资金池划转到销毁账户（burn）。

> 以上资金分配应在 `settled` 时生成结算单；实际出池发生在“提取接口”，并且必须幂等。

### 幂等约束（接口级）
为避免重复扣款/重复出池，每个写接口需要支持幂等：
- 加入赌局、庄家加注、提取、庄家宣布、挑战者异议、管理员裁决：均应携带 `request_id`（或 Idempotency-Key）。
- 服务端以（battle_id + user_id + action + request_id）作为幂等键进行去重。

每次评论 或者点赞 都会增加对应的 实体以及实体ID的热度
