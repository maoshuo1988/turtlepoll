# 对立PK：规则与计算

## 回合制规则

每个 PK 话题持续存在，系统按回合推进。

| 阶段 | 时间窗口 | 允许操作 | 状态 |
|------|----------|----------|------|
| 下注期 | 第 0-48 小时 | 下注、评论、点赞、回复、拉踩 | `betting` |
| 锁局期 | 第 48-72 小时 | 评论、点赞、回复、拉踩 | `locked` |
| 冷却期 | 结束后 10 分钟 | 不允许用户动作 | `cooldown` |

建议后端存储：

- `roundStartTime`
- `lockTime = roundStartTime + 48h`
- `roundEndTime = roundStartTime + 72h`
- `nextRoundTime = roundEndTime + 10min`
- `phase`

阶段可以冗余存储，也可以由时间实时计算；为方便查询和定时任务，建议冗余存储并定时校正。

## 下注与赔率

### 下注规则

- 每票固定消耗 100 龟币。
- 每个用户每个回合只能下注一次。
- 用户下注后阵营锁定，不能换边。
- 只有 `betting` 阶段允许下注。
- 下注本金进入 PK 资金池。

### 奖池规则

每局维护：

- `poolA`: A方下注总额。
- `poolB`: B方下注总额。
- `betCountA`: A方下注人数。
- `betCountB`: B方下注人数。

结算时：

- 胜方用户取回本金。
- 胜方按自己下注额占胜方总下注额的比例瓜分败方奖池。
- 固定票价时，等价于胜方按人数均分败方奖池。

### 动态赔率

规则文档描述“赔率根据双方投票人数动态计算”。

建议展示赔率：

```text
oddsA = (poolA + poolB) / max(poolA, 1)
oddsB = (poolA + poolB) / max(poolB, 1)
```

如果希望避免一边无人导致展示过高，可以加入虚拟底池：

```text
effectiveA = virtualPool + poolA
effectiveB = virtualPool + poolB
oddsA = (effectiveA + effectiveB) / effectiveA
oddsB = (effectiveA + effectiveB) / effectiveB
```

P0 建议只用于展示，结算仍按真实奖池分配。

## 热度计算

总热度：

```text
Heat_total = H_like + H_comment + H_downvote + H_bet
```

### 点赞热度

```text
Like_i = 1 * U_i * L_decay_i * AntiSpam_i
L_decay_i = 1 / (1 + 0.2 * (k - 1))
H_like = sum(Like_i)
```

P0 简化建议：

- 先按有效点赞数计入。
- 用户权重、频率衰减、AntiSpam 在 P1 加入。

### 评论热度

基础分：

```text
Base_i = 3 * A_i * U_i * T_i * C_i
A_i = 1 / (1 + 0.5 * (n - 1))
```

互动热度：

```text
H_i = log(1 + L_i + 1.5 * R_i)
Comment_i = Base_i * (1 + H_i * Q_i * F_i)
Comment_i_final = min(Comment_i, 20)
```

P0 简化建议：

- 评论基础分固定 3。
- 回复按 1.5 权重。
- 点赞使用 `log(1 + likeCount)`。
- 单条上限 20。

### 拉踩热度

```text
Downvote_i = 2 * U_i * D_decay_i * AntiSpam_i
D_decay_i = 1 / (1 + 0.3 * (k - 1))
H_downvote = sum(Downvote_i)
```

对目标评论影响：

```text
Comment_i = Comment_i * (1 / (1 + 0.2 * Downvotes))
```

P0 可以先不做拉踩；如果做，必须有唯一约束防止同一用户重复踩同一评论。

### 下注热度

```text
Bet_i = sqrt(Bet_amount_i) * 0.5 * U_i
H_bet = sum(Bet_i)
```

下注会消耗龟币，建议复用 `UserCoinService.SpendToPool`。P0 固定每票 100 龟币时，也可以简化为每次有效下注贡献固定下注热度。

## 胜负结算

回合结束时执行：

1. 锁定回合，停止所有用户动作。
2. 计算 A/B 最终热度。
3. 热度高的一方为胜方。
4. 写入 `PKRound.winner`、`heatA`、`heatB`。
5. 生成每个下注用户的结算明细。
6. 更新话题总战绩与连胜。
7. 更新当前赛季胜场。
8. 进入冷却期。

### 平局口径

原规则未说明平局。

建议 P0 选择一种：

- `draw_refund`: 平局退还双方本金。
- `draw_keep`: 平局按上一局胜方守擂。

更稳妥的是 `draw_refund`，避免争议。

## 赛季与历史

### 赛季

- 每个赛季 30 天。
- 一个赛季约 10 局。
- 赛季内统计 A/B 胜场。
- 赛季结束产生冠军。
- 自动开启下一赛季。

### 总战绩

每个话题需要维护：

- 总局数。
- A方总胜场。
- B方总胜场。
- A/B 总胜率。
- A/B 最高连胜。
- 当前连胜方与连胜数。

总战绩可由 `PKRound` 计算得出，但为了列表性能，建议在 `PKTopicStats` 或 `PKTopic` 冗余关键字段。

### 守擂与翻盘

展示逻辑：

- 上一局胜方当前仍领先：守擂中。
- 上一局胜方当前落后：被翻盘。
- 无上一局时不展示。

该状态可实时由上一局 winner 与当前 `currentHeatA/currentHeatB` 派生，不一定落库。
