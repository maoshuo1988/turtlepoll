# 对立PK：流程图

## 赛季流程

```mermaid
flowchart TD
  A[创建或启用PK话题] --> B{是否存在当前赛季}
  B -->|否| C[创建第1个赛季<br/>周期30天]
  B -->|是| D[读取当前赛季]
  C --> E{是否存在当前局}
  D --> E
  E -->|否| F[创建赛季第1局]
  E -->|是| G[进入当前局流程]
  F --> G

  G --> H[每局结束后写入胜方]
  H --> I[更新赛季胜场<br/>winsA/winsB/totalRounds]
  I --> J{赛季是否到期}
  J -->|否| K[继续创建下一局]
  K --> G
  J -->|是| L[计算赛季冠军<br/>A/B/draw]
  L --> M[归档当前赛季为 finished]
  M --> N[创建下一赛季]
  N --> O[创建新赛季第1局]
  O --> G
```

关键落点：

- 赛季实体：`PKSeason`。
- 当前赛季引用：`PKTopic.currentSeasonId`。
- 赛季推进入口：`PKService.CronTick()`。
- 赛季冠军依据：赛季内 A/B 胜场。

## 每一局PK的流程

```mermaid
flowchart TD
  A[创建PKRound] --> B[phase=betting<br/>下注期0-48小时]

  B --> C[用户下注<br/>每人每局一次]
  B --> D[评论/点赞/回复/拉踩]
  C --> E[更新奖池/下注人数/下注热度]
  D --> F[更新互动热度]
  E --> G[刷新A/B实时热度]
  F --> G

  G --> H{到达lockTime?}
  H -->|否| B
  H -->|是| I[phase=locked<br/>锁局期48-72小时]

  I --> J[禁止下注]
  I --> K[允许评论/点赞/回复/拉踩]
  K --> L[持续更新互动热度]
  L --> M[刷新A/B实时热度]

  M --> N{到达endTime?}
  N -->|否| I
  N -->|是| O[锁定本局]
  O --> P[计算最终heatA/heatB]
  P --> Q{比较热度}
  Q -->|A更高| R[winner=A]
  Q -->|B更高| S[winner=B]
  Q -->|相等| T[winner=draw]

  R --> U[生成结算明细]
  S --> U
  T --> U
  U --> V[更新总战绩/连胜/赛季胜场]
  V --> W[phase=cooldown<br/>冷却10分钟]
  W --> X{到达nextRoundTime?}
  X -->|否| W
  X -->|是| Y[创建下一局]
```

关键落点：

- 单局实体：`PKRound`。
- 用户下注：`PKBet`。
- 评论归属：`PKCommentMeta`。
- 热度来源：`PKAction`。
- 结算明细：`PKSettlementItem`。

## 用户获利的流程

```mermaid
flowchart TD
  A[用户进入PK详情] --> B{当前是否下注期}
  B -->|否| C[不可下注<br/>只能查看或互动]
  B -->|是| D[选择A/B阵营下注]

  D --> E{用户本局是否已下注}
  E -->|是| F[返回已有下注记录<br/>不重复扣款]
  E -->|否| G[扣除100龟币]
  G --> H[下注本金进入PK资金池]
  H --> I[创建PKBet]
  I --> J[等待本局结束]

  J --> K[系统计算A/B最终热度]
  K --> L{本局结果}

  L -->|用户阵营获胜| M[计算用户收益]
  M --> N[返还本人本金]
  N --> O[按占比分配败方奖池]
  O --> P[生成派奖金额 payout]
  P --> Q[资金池出账到用户金币账户]
  Q --> R[写入金币流水 PK_PAYOUT]

  L -->|用户阵营失败| S[本金进入败方奖池]
  S --> T[payout=0]
  T --> U[写入结算结果 lose]

  L -->|平局| V[退还本人本金]
  V --> W[资金池出账到用户金币账户]
  W --> X[写入金币流水 PK_DRAW_REFUND]

  R --> Y[用户查看我的PK记录]
  U --> Y
  X --> Y
```

收益口径：

```text
胜方用户收益 = 本人下注本金 + 败方奖池 * (本人下注额 / 胜方总下注额)
失败方用户收益 = 0
平局用户收益 = 本人下注本金
```

P0 固定每人每局下注 100 龟币时，胜方也可以理解为按胜方人数均分败方奖池。

## 事务与幂等要求

- 下注接口必须幂等：同一用户同一局重复请求不能重复扣款。
- 结算任务必须幂等：同一回合只能生成一次结算明细。
- 派奖必须幂等：同一用户同一局只能入账一次。
- 下一局创建必须幂等：同一话题的 `roundNo` 不能重复。
- 赛季创建必须幂等：同一话题的 `seasonNo` 不能重复。
