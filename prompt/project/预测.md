1.基于用户金币设计如下系统
2.管理员可以对账户进行铸币
3.用户可以使用金币对预测市场下注
4.每个预测市场都有两个选项A\B，并且要记录这两个选项所有用户的下注作为奖金池
5.系统默认投放的A/B筹码 默认500
6.赔率为了避免前端显示极端 1.01 或 100 这种数值，给赔率加上下限：1.2-5.0；
7.示例：早期有虚拟底池时的表现

假设：

```text
baseA = 500, baseB = 500
VA_user = 0, VB_user = 0
```

初始时：

```text
VA_eff = 500
VB_eff = 500
T_eff  = 1000

oddsA_raw = 1000 / 500 = 2.0
oddsB_raw = 1000 / 500 = 2.0
→ 初始赔率：2.0 / 2.0
```

第一个用户押 A 100：

```text
VA_user = 100, VB_user = 0
VA_eff = 600, VB_eff = 500, T_eff = 1100

oddsA_raw ≈ 1100 / 600 ≈ 1.83
oddsB_raw = 1100 / 500 = 2.20
→ A 端略变热门，B 端略变冷门
```

再有人押 B 400：

```text
VA_user = 100, VB_user = 400
VA_eff = 600, VB_eff = 900, T_eff = 1500

oddsA_raw = 1500 / 600 = 2.50
oddsB_raw = 1500 / 900 ≈ 1.67
→ B 端资金更多，赔率更低；A 端资金相对少，赔率被拉高
8.下注时锁赔率、不在结算时按最新赔率重算
