# 身份对立PK - 完整规则文档

## 一、模块概述

身份对立PK是龟投平台的核心玩法模块之一，专注于**永恒话题的对立阵营PK对决**。每个话题代表一个永恒的争议（如"梅西 vs C罗"），采用**回合制**进行持续对抗，通过**热度计算系统**判定每局胜负。

---

## 二、核心机制

### 2.1 话题特性
- 每个PK话题是**永恒存在**的，不会结束
- 话题双方代表两个对立阵营（A方 vs B方）
- 用户选择阵营后进行投票、评论、点赞、拉踩等操作

### 2.2 回合制规则

每一局 **3天（72小时）**，分为三个阶段：

| 阶段 | 时间 | 允许的操作 | 状态标识 |
|------|------|-----------|---------|
| **下注期** | 第1-2天（0-48小时） | 下注、评论、点赞、拉踩、投币 | 🟢 下注中 |
| **锁局期** | 第3天（48-72小时） | 评论、点赞、拉踩（**不可下注**） | 🟡 锁局中 |
| **冷却期** | 本局结束后10分钟 | 无操作，等待下一局 | 🔵 冷却中 |

### 2.3 胜负判定
- 每局结束时，按**热度公式**计算双方阵营总热度
- **热度高的一方为本局胜者**
- 胜方全体投票者按赔率瓜分败方投入的龟币奖池

### 2.4 下注规则
- 每票消耗 **100 龟币**
- 每个用户每局只能投票一次（选择一方后锁定）
- 锁局期（第3天）不可再下注
- 赔率根据双方投票人数动态计算

## 三、热度计算公式

### 3.1 总热度公式

```
Heat_total = H_like + H_comment + H_downvote + H_coin
```

每个阵营的总热度为：点赞热度 + 评论热度 + 拉踩热度 + 投币热度 之和。

### 3.2 全局变量

| 符号 | 含义 | 范围 |
|------|------|------|
| U | 用户权重 | 0.3 - 1.2 |
| AntiSpam | 反作弊系数 | 0 - 1 |
| k | 用户本局第k次操作 | ≥1 |
| n | 用户第n条评论 | ≥1 |
| m | 用户在同一评论下第m条回复 | ≥1 |

### 3.3 用户权重(U)

| 用户类型 | 权重(U) |
|---------|---------|
| 新用户 | 0.8 |
| 普通用户 | 1.0 |
| 活跃用户 | 1.2 |
| 风险用户 | 0.3 |

### 3.4 点赞热度 (H_like)

```
Like_i = 1 × U_i × L_decay_i × AntiSpam_i
H_like = Σ Like_i
```

| 参数 | 定义 | 默认值/公式 |
|------|------|-----------|
| 基础权重 | 每次点赞基础值 | 1 |
| U_i | 用户权重 | 见用户权重表 |
| L_decay_i | 频率衰减 | `1 / (1 + 0.2 × (k-1))` |
| AntiSpam_i | 风控调整 | 0-1 |

### 3.5 评论热度 (H_comment)

**基础分：**
```
Base_i = 3 × A_i × U_i × T_i × C_i
```

| 参数 | 定义 | 公式/值 |
|------|------|--------|
| 3 | 评论基础权重 | 3 |
| A_i | 评论顺序衰减 | `1 / (1 + 0.5 × (n-1))` |
| U_i | 用户权重 | 0.3-1.2 |
| T_i | 时间权重 | 正常1.0 / 最后10分钟0.8 / 最后3分钟0.5 |
| C_i | 内容质量 | 1.0 / 0.6 / 0.2 / 0 |

**互动热度：**
```
H_i = log(1 + L_i + 1.5 × R_i)
```

| 参数 | 定义 | 说明 |
|------|------|------|
| L_i | 有效点赞数 | 加权求和 |
| R_i | 有效回复数 | 加权+衰减 |
| 1.5 | 回复乘数 | 回复价值 > 点赞 |
| log | 自然对数 | 防止数值爆炸 |

**最终评论分：**
```
Comment_i = Base_i × [1 + H_i × Q_i × F_i]
Comment_i_final = min(Comment_i, Cap_comment)
```

| 参数 | 定义 | 默认值 |
|------|------|--------|
| Q_i | 质量修正 | 0.7-1.2 |
| F_i | 反作弊修正 | 0-1 |
| Cap_comment | 单条评论上限 | 20 (MVP) |

```
H_comment = Σ Comment_i_final
```

### 3.6 拉踩热度 (H_downvote)

```
Downvote_i = 2 × U_i × D_decay_i × AntiSpam_i
H_downvote = Σ Downvote_i
```

| 参数 | 定义 | 公式 |
|------|------|------|
| 2 | 基础权重 | 2 |
| D_decay | 频率衰减 | `1 / (1 + 0.3 × (k-1))` |

**对目标评论的影响：**
```
Comment_i ← Comment_i × (1 / (1 + 0.2 × Downvotes))
```

### 3.7 投币热度 (H_coin)

```
Coin_i = sqrt(Coin_amount_i) × 0.5 × U_i
H_coin = Σ Coin_i
```

| 参数 | 定义 | 说明 |
|------|------|------|
| sqrt | 平方根缩放 | 防止大户碾压 |
| 0.5 | 缩放系数 | 可配置 |

### 3.8 反作弊系统

AntiSpam 系数 ∈ [0, 1]

| 等级 | 系数值 |
|------|--------|
| 正常 | 1.0 |
| 轻度异常 | 0.8 |
| 可疑 | 0.5 |
| 严重刷量 | 0.2 |

检测信号：账号年龄、操作频率、重复内容、IP/设备聚类

## 四、赛季系统

### 4.1 赛季规则
- 每个赛季 **30天**
- 赛季内进行约 **10局** PK
- 赛季结束后统计双方胜场数
- 胜场多的一方获得 **赛季冠军** 🏆

### 4.2 赛季记录
- 赛季编号、起止日期
- 双方胜场数
- 赛季冠军归属

### 4.3 赛季间续接
- 赛季结束后自动开启下一赛季
- 历史赛季记录永久保存

---

## 五、历史战绩

每个PK话题记录以下历史数据：

### 5.1 总战绩
- 总局数
- A方总胜场 / B方总胜场
- 双方总胜率

### 5.2 连胜记录
- A方最高连胜
- B方最高连胜
- 当前连胜方和连胜数

### 5.3 赛季历史
- 每个赛季的详细战绩
- 赛季冠军归属

### 5.4 对局记录
- 每一局的热度对比（A方热度 vs B方热度）
- 评论数、点赞数等互动数据
- 胜负结果

## 六、守擂与翻盘机制

### 6.1 守擂提示
- 上一局胜方在当前局仍然领先
- 显示绿色标识：**"守擂中 🛡️"**

### 6.2 翻盘提示
- 上一局胜方在当前局落后
- 显示红色标识：**"被翻盘 ⚡"**

### 6.3 下一局开局提示
- 冷却期结束前显示：上局胜方名称 + "守擂" 或 "翻盘"提示
- 激发用户参与下一局的动力

---

## 七、评论战场

### 7.1 功能说明
- 每个PK话题都有独立的评论战场
- 评论分为A方阵营和B方阵营，双列显示
- 评论内容直接影响热度计算

### 7.2 互动功能
- **评论**：为己方阵营贡献评论热度
- **点赞**：增加评论的互动热度
- **丢💩（拉踩）**：降低对方评论热度，同时为己方增加拉踩热度
- **回复**：支持嵌套回复，回复价值高于点赞（1.5倍乘数）

### 7.3 视觉效果
- 龟势PK火焰文字特效
- 双方对抗条（蓝/红渐变 + VS球 + 弹簧动画）
- 粒子特效背景
- MVP头像展示（点赞最多的评论者）
- 评论发送后触发对抗条震动

## 九、技术实现要点（后端参考）

### 9.1 数据结构

**PK话题状态 (PKTopicState)**
- `id`: 话题ID
- `currentRound`: 当前局数
- `phase`: 当前阶段 (`betting` | `locked` | `cooldown`)
- `roundStartTime`: 本局开始时间 (unix ms)
- `roundEndTime`: 本局结束时间 (unix ms)
- `lockTime`: 锁局时间 (unix ms, roundStart + 48h)
- `nextRoundTime`: 下一局开始时间 (roundEnd + 10min)
- `currentHeatA` / `currentHeatB`: 当前局双方实时热度
- `roundHistory`: 历史对局记录数组
- `season`: 当前赛季信息
- `history`: 全部历史战绩

**对局记录 (PKRoundResult)**
- `round`: 局数
- `heatA` / `heatB`: 双方最终热度
- `winner`: 胜方 (`A` | `B`)
- `betCountA` / `betCountB`: 双方下注人数
- `commentCount` / `likeCount`: 互动数据

**赛季 (PKSeason)**
- `season`: 赛季编号
- `startDate` / `endDate`: 起止日期
- `totalRounds`: 总局数
- `winsA` / `winsB`: 双方胜场
- `champion`: 赛季冠军 (`A` | `B` | null进行中)

### 9.2 定时任务
1. **每局结束 (roundEndTime)**：计算双方热度 → 判定胜负 → 记录结果 → 进入冷却
2. **冷却结束 (nextRoundTime)**：重置热度 → 开启新一局 → 设置新的时间窗口
3. **赛季结束 (30天)**：统计赛季冠军 → 开启新赛季

### 9.3 API接口建议

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/pk/topics` | GET | 获取所有PK话题列表（含当前状态） |
| `/api/pk/topic/{id}` | GET | 获取单个PK话题详情 |
| `/api/pk/bet` | POST | 下注 `{topicId, side, amount}` |
| `/api/pk/history/{id}` | GET | 获取历史战绩 |
| `/api/pk/season/{id}` | GET | 获取赛季记录 |
| `/api/pk/heat/{id}` | GET | 获取当前实时热度 |

## 十、完整热度公式汇总

```
Heat_total =
  Σ(1 × U × L_decay × AntiSpam)                                    // 点赞
  + Σ(min(Base × [1 + log(1+L+1.5R) × Q × F], 20))                // 评论
  + Σ(2 × U × D_decay × AntiSpam)                                  // 拉踩
  + Σ(sqrt(CoinAmount) × 0.5 × U)                                  // 投币

其中:
  Base = 3 × A × U × T × C
  L_decay = 1/(1+0.2×(k-1))
  D_decay = 1/(1+0.3×(k-1))
  A = 1/(1+0.5×(n-1))
  T = 1.0/0.8/0.5 (按时间阶段)
  C = 1.0/0.6/0.2/0 (按内容质量)
```
